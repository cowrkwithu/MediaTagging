## 3. 기능 요구사항

### 3.1 동영상 업로드 및 메타데이터 생성

#### FR-001: 동영상 업로드
- 사용자는 동영상 파일을 시스템에 업로드할 수 있어야 한다.

#### FR-002: 자동 메타데이터 생성
- 시스템은 업로드된 동영상에 대해 다음 정보를 자동 생성해야 한다:
  - 동영상 파일명
  - 동영상 전체에 대한 요약
  - 동영상에 대한 태그

#### FR-003: 사용자 입력 메타데이터
- 사용자는 다음 정보를 직접 입력할 수 있어야 한다:
  - 동영상 제목
  - 동영상에 대한 기록(메모)

---

### 3.2 영상 태깅 작업

#### FR-004: 장면 단위 분할
- 사용자가 저장 후 영상 태깅 작업을 시작하면, 시스템은 영상을 장면 단위로 분할해야 한다.

#### FR-005: 장면별 타임스탬프 생성
- 각 장면에 대해 타임스탬프를 생성해야 한다.

#### FR-006: 장면별 태그 생성
- 각 장면에 대해 1개 이상 10개 이하의 태그를 생성해야 한다.

---

### 3.3 장면 단위 영상 저장

#### FR-007: 장면 선택
- 사용자는 장면 단위로 나뉘어진 영상들 중 원하는 장면을 선택할 수 있어야 한다.

#### FR-008: 개별 영상 저장
- 선택한 장면을 별도의 영상 파일로 저장할 수 있어야 한다.

---

### 3.4 키워드 검색

#### FR-009: 키워드 검색
- 사용자는 키워드를 이용하여 영상을 검색할 수 있어야 한다.

#### FR-010: 논리 연산자 지원
- 검색 시 다음 논리 연산자를 지원해야 한다:
  - AND (그리고): 모든 키워드를 포함하는 결과
  - OR (또는): 키워드 중 하나 이상을 포함하는 결과
  - NOT (제외): 특정 키워드를 제외한 결과

#### FR-011: 검색 대상
- 검색은 다음을 대상으로 수행되어야 한다:
  - 동영상 전체 태그
  - 장면별 태그
  - 동영상 제목
  - 동영상 요약

---

### 3.5 동영상 제작 요청

#### FR-012: 영상 선택
- 사용자는 검색 결과에서 다음을 선택할 수 있어야 한다:
  - 장면 단위로 나뉘어진 영상
  - 원본 영상

#### FR-013: 외부 API 연동
- 선택한 영상을 Google Nanobana API를 이용하여 동영상 제작을 요청할 수 있어야 한다.

---

## 4. 사용자 워크플로우

```
1. 동영상 업로드
       │
       ▼
2. 자동 메타데이터 생성 (파일명, 요약, 태그)
       │
       ▼
3. 사용자 메타데이터 입력 (제목, 기록) [선택사항]
       │
       ▼
4. 저장
       │
       ▼
5. 영상 태깅 작업 시작
       │
       ▼
6. 장면 단위 분할 + 타임스탬프 + 태그 생성
       │
       ▼
7. 장면 선택 및 개별 영상 저장 [선택사항]
       │
       ▼
8. 키워드 검색 (논리 연산자 활용)
       │
       ▼
9. 검색 결과에서 영상 선택
       │
       ▼
10. Google Nanobana API를 통한 동영상 제작 요청
```

## 5. 비기능 요구사항

### NFR-001: 웹 접근성
- 주요 웹 브라우저(Chrome, Firefox, Safari, Edge)에서 정상 동작해야 한다.

### NFR-002: 사용자 인터페이스
- 직관적이고 사용하기 쉬운 인터페이스를 제공해야 한다.

### NFR-003: 응답 시간
- 동영상 업로드 및 처리 상태를 사용자에게 실시간으로 표시해야 한다.

## 6. 용어 정의

| 용어 | 정의 |
|------|------|
| 장면(Scene) | 동영상 내에서 의미적으로 구분되는 단위 영상 구간 |
| 태그(Tag) | 영상 또는 장면의 내용을 설명하는 키워드 |
| 타임스탬프(Timestamp) | 장면의 시작 및 종료 시점을 나타내는 시간 정보 |
| 메타데이터(Metadata) | 동영상에 대한 부가 정보 (제목, 요약, 태그 등) |
